PRIVACY POLICY

All user information collected by the Sylphy Bot and the Reference Assistant bot is accessible by any Discord user.

By using the Bots, inviting the Bots to your server, using a server that the Bots have joined, or in some other way agreeing to the Sylphy Bot and Reference Assistant Terms of Service, you are also accepting this privacy policy and agreement. This privacy policy is written to outline the kind of data that the Bots collect and store, and also how this data is used.

Data we collect:

1. User ID: The identification number attached to all user accounts. User IDs are accessible to anyone with a Discord account, and can be seen after enabling Developer Mode in the Discord settings. The Sylphy Bot stores User IDs in order to identify the artists ("sketchers") of the images contained in the Sylphy Bot search library. The Sylphy Bot collects User IDs automatically from embedded messages sent by the Karuta Bot. The Reference Assitant bot stores User IDs in order to provide a user-specific image history library. The Reference Assistant bot collects User IDs automatically when users issue commands to the Assistant.

2. Channel ID: The identification number attached to all guild channels. Channel IDs are accessible to anyone with a Discord account, and can be seen after enabling Developer Mode in the Discord settings. We store Channel IDs to flag specific channels as operable as a "display channel" (for the Sylphy Bot) or as a "conversion channel" (for the Reference Assistant). The Sylphy Bot will automatically display into flagged "display channels" any new entry to the Sylphy Bot's search library as they are entered. The Reference Assistant bot will only convert images if they are posted into a flagged "conversion channel. We store Channel IDs only at the request of the Guild owner(s) for any guild to which the stored channel belongs, and they are only stored manually.

3. Message ID: The identification number attached to all Discord messages. Message IDs are accessible to anyone with a Discord account, and can be seen after enabling Developer Mode in the Discord settings. We store Message IDs for use in file names for the images sent by the Reference Assistant. Using Message IDs in file names ensures no duplicate file name occurs.

4. Message Content and Embeds: The Sylphy Bot updates its library by reading "Card" data from Karuta Bot's embedded messages. The data read from these embedded messages includes "Series Names", "Character Names", "Character Aliases", "Card IDs", "Print Numbers", "Edition Numbers", "Card Ink", "Card Inkwells", "Sketcher IDs", and the embedded messages' thumbnail images. This data is accessible to all verified users of the Karuta Bot. This data is only obtained from the Karuta Bot's messages. We analyze Message Content from the Sylphy Bot's users' messages to process text commands, but we do not store any non-bot user message content. The Reference Assistant accesses the images attached to user messages sent in a flagged "conversion channel" in order to analyze the image and reduce the color palette of the image. The original image sent by the user is not stored by the Reference Assistant; only the converted image is stored.


We do not sell or share any collected data with any third parties.

 

DATA DELETION

Users may request at any time for us to remove their User ID or Channel IDs by joining the support server linked in the Sylphy Bot's and the Reference Assistant's profiles. However, all User IDs stored in the Sylphy Bot are collected from the Karuta Bot's messages, not from the users themselves. Unless a user first requests their data deletion from the Karuta Bot, removing their data from the Sylphy Bot will only temporarily remove their User ID, and the User ID is at risk of being recorded again unless the user requests data deletion from the Karuta Bot first.
